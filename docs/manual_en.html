1. Overview

Rocuronium PK/PD Simulator is a pharmacokinetic/pharmacodynamic simulation tool designed for educational and research use. It predicts plasma concentration (Cp), effect-site concentration (Ce), and Train-of-Four (TOF) ratio of rocuronium based on three-compartment pharmacokinetic models and a pharmacodynamic model.

The application provides a straightforward simulation workflow through four tabs:

  • Patient -- Configure patient demographics and select a PK/PD model.
  • Dosing -- Define a dosing schedule with bolus and continuous infusion events.
  • Results -- View simulated concentration-time curves, TOF ratio, and tabular data.
  • Info -- Application information, version details, and references.

Five published PK models are available (Wierda, Szenohradszky, Cooper, Alvarez-Gomez, McCoy), combined with the PD model by Masui et al. (2018). The simulator uses enhanced RK4 (Runge-Kutta 4th order) numerical integration for accurate concentration and effect predictions.

Not for Clinical Use This application is strictly for educational and research purposes. It must not be used as a basis for clinical diagnosis or treatment decisions. All clinical decisions must be made under the responsibility of qualified medical professionals. Calculation results are theoretical values.

2. Getting Started

2.1 Installation (PWA)

The application is a Progressive Web App (PWA). Once loaded in a browser, it can be installed to your home screen and used offline. This is particularly useful for iPhone and iPad users in environments without reliable network access.

To install on iPhone / iPad (Safari):

  1. Open the application URL in Safari.
  2. Tap the Share button (square with upward arrow) in the browser toolbar.
  3. Scroll down and tap Add to Home Screen.
  4. Tap Add to confirm. The app icon will appear on your home screen.

After installation, the app launches in full-screen mode without browser navigation bars and works entirely offline.

2.2 Disclaimer Screen

When the application starts, a disclaimer modal is displayed. You must read and accept the terms before proceeding.

Tap Accept and Start to proceed to the main application.

3. Dosing Schedule

The Dosing tab allows you to define a sequence of dose events that describe the rocuronium administration schedule. Each event specifies a point in time with an optional bolus dose and/or a continuous infusion rate change.

Figure 4: Dosing tab with default dose event

3.1 Adding Dose Events

A default initial dose event is pre-populated when the app starts. To add additional dose events:

  1. Tap the + Add Dose Event button on the Dosing tab.
  2. In the Add Dose Event modal, set the Time (minutes from anesthesia start or clock time).
  3. Enter a Bolus amount in mg (0 -- 100 mg). Set to 0 if no bolus is given at this time point.
  4. Enter the Continuous infusion rate in μg/kg/min (0 -- 30 μg/kg/min). Set to 0 if no continuous infusion.
  5. Tap Add to save the dose event.

Figure 5: Add Dose Event modal

3.2 Dose Parameters

Parameter Unit Range Description
Time minutes / HH:MM From anesthesia start Time point at which the dose event occurs
Bolus mg 0 -- 100 Single bolus dose of rocuronium (total amount, not weight-based)
Continuous μg/kg/min 0 -- 30 Continuous infusion rate (weight-based, per minute)
Note on Units The bolus dose is specified in mg (total dose), while the continuous infusion rate is specified in μg/kg/min (weight-based rate per minute). This reflects standard clinical practice for rocuronium administration.

3.3 Stopping Dosing

To simulate the cessation of a continuous infusion, add a dose event at the desired stop time with both bolus and continuous rate set to 0. This instructs the simulator to halt all drug delivery from that time point onward, allowing the model to predict the decay of plasma and effect-site concentrations and the recovery of TOF ratio.

Existing dose events can be removed by tapping the delete button associated with each event entry.

4. Simulation Results

After configuring patient parameters and dose events, navigate to the Results tab to run the simulation and view the predicted concentration-time profiles and neuromuscular blockade.

4.1 Summary Cards

The top of the Results tab displays summary cards showing key simulation outputs:

  • Plasma Concentration (Cp) -- Peak predicted plasma concentration in μg/mL.
  • Effect-site Concentration (Ce) -- Peak predicted effect-site concentration in μg/mL.
  • TOF Ratio -- Minimum predicted Train-of-Four ratio as a percentage. A TOF ratio of 0% indicates complete neuromuscular blockade; 100% indicates full recovery.

4.2 Concentration Chart

The concentration chart is the primary visualization of the simulation results. It plots three curves over time:

  • Plasma Concentration (Cp) -- Shown on the left y-axis (μg/mL).
  • Effect-site Concentration (Ce) -- Shown on the left y-axis (μg/mL).
  • TOF Ratio -- Shown on the right y-axis (%).

The dual y-axis design allows simultaneous visualization of drug concentrations and their pharmacodynamic effect (neuromuscular blockade). The x-axis represents elapsed time from anesthesia start.

Figure 6: Results tab with concentration and TOF ratio chart

4.3 Data Table

Below the chart, a data table provides the numerical simulation output at each time step. The table includes columns for time, plasma concentration, effect-site concentration, and TOF ratio. Scroll down to view the full dataset.

Figure 7: Results data table

Tip The chart uses the enhanced RK4 (Runge-Kutta 4th order) integration method introduced in V1.1.0 for improved numerical accuracy, particularly during rapid concentration changes following bolus injections.

5. CSV Export

The simulation data can be exported as a CSV file for further analysis in spreadsheet software, statistical tools, or graphing applications.

  1. Navigate to the Results tab and ensure the simulation has been run.
  2. Tap the Export CSV button.
  3. The CSV file is generated and downloaded. It includes timestamped columns for plasma concentration, effect-site concentration, and TOF ratio.

The exported file contains the complete time-series data at each simulation time step, suitable for creating publication-quality figures or performing additional statistical analysis.

Tip On iOS, the CSV file can be saved to the Files app or shared directly via AirDrop, email, or other share sheet options.

6. PK/PD Models Reference

The simulator includes five published three-compartment pharmacokinetic models for rocuronium, combined with a pharmacodynamic model for TOF ratio prediction.

Pharmacokinetic Models

Model Reference Description
Wierda Wierda et al. Three-compartment PK model for rocuronium. One of the earliest and most widely cited models for rocuronium pharmacokinetics.
Szenohradszky Szenohradszky et al. Three-compartment model with parameters derived from a study population that characterized rocuronium distribution and elimination kinetics.
Cooper Cooper et al. PK model incorporating compartmental analysis of rocuronium disposition, providing volume and clearance parameters for simulation.
Alvarez-Gomez Alvarez-Gomez et al. Three-compartment model derived from clinical pharmacokinetic data, reflecting rocuronium behavior across different patient populations.
McCoy McCoy et al. PK model characterizing rocuronium pharmacokinetics with emphasis on compartmental volumes and inter-compartmental clearances.

Pharmacodynamic Model

All PK models are paired with the pharmacodynamic model by Masui et al. (2018), published in the Journal of Anesthesia (volume 32, pages 709-716). This PD model relates the effect-site concentration of rocuronium to the Train-of-Four (TOF) ratio using a sigmoid E-max relationship, enabling prediction of the degree of neuromuscular blockade over time.

Tip Different PK models may produce different concentration-time profiles for the same dosing regimen. Comparing results across models can provide insight into the range of predicted responses and model-specific assumptions.

8. Tips and Best Practices

Patient Parameters

  • Always set patient parameters before defining dose events and running the simulation. PK model calculations depend on weight (and potentially age, sex, and height depending on the selected model).
  • The continuous infusion rate is weight-based (μg/kg/min), so changing the patient weight will affect the total drug delivered per unit time.
  • Use the range sliders for quick adjustment of age, weight, and height values.

Model Selection

  • Different PK models were developed from different study populations and may yield varying predictions. Consider the clinical context when selecting a model.
  • The Wierda model is set as the default and is one of the most commonly referenced models for rocuronium PK.
  • Running the same dosing scenario across multiple models can help illustrate the uncertainty inherent in pharmacokinetic predictions.

Dosing Strategy

  • A typical intubation dose of rocuronium is 0.6 -- 1.2 mg/kg. For a 70 kg patient, this corresponds to approximately 42 -- 84 mg.
  • Maintenance continuous infusion rates typically range from 5 -- 15 μg/kg/min, adjusted based on the desired level of neuromuscular blockade.
  • To simulate recovery, add a final dose event with both bolus and continuous rate set to 0 at the desired stop time.

Interpreting TOF Ratio

  • TOF ratio approaching 0% indicates deep neuromuscular blockade (no twitch response).
  • TOF ratio returning to 90% or above is generally considered adequate recovery from neuromuscular blockade.
  • The relationship between effect-site concentration and TOF ratio follows a sigmoid curve, meaning small changes in concentration near the EC50 produce large changes in TOF ratio.

Offline Use

  • After installing as a PWA, the app works fully offline. No network connection is needed during use.
  • CSV export saves files locally to the device.

Screen Orientation

The app is optimized for portrait orientation on mobile devices. Charts and controls are designed for single-hand operation on smartphones.

9. Troubleshooting

The app does not load or shows a blank screen

Ensure you are using a supported browser (Safari on iOS, Chrome or Edge on desktop). Clear the browser cache and reload the page. If using the PWA, try removing and reinstalling the app.

The chart does not display after adding dose events

Confirm that you have navigated to the Results tab and that the simulation has been executed. Ensure at least one dose event with a non-zero bolus or continuous rate is defined.

Concentration values seem incorrect

Verify that patient parameters (especially weight) are correctly set, as the continuous infusion rate is weight-based. Also check that the correct PK model is selected. Different models will produce different concentration profiles for the same dosing schedule.

TOF ratio does not change

If the simulated concentrations are very low (well below the EC50 of the PD model), the TOF ratio may remain near 100%. Conversely, at very high concentrations, the TOF ratio may remain near 0%. Ensure the dosing regimen produces concentrations in the pharmacodynamically active range.

CSV export does not work

On iOS, the file download behavior may differ depending on the browser. In Safari, the file may open in a new tab or prompt a save dialog. Try using the Share sheet to save to the Files app.

Range sliders are difficult to adjust precisely

On small screens, the range sliders may require careful touch placement. Try tapping the desired position on the slider track rather than dragging, or use the displayed numerical value to confirm the current setting.
